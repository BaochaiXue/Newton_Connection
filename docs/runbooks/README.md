# Runbooks

This directory stores reusable workflow instructions that are broader than a
single task but more operational than architecture docs.

Examples:

- how to create a new experiment directory
- how to promote a tmp result into a documented artifact
- how to prepare meeting-ready outputs
- how to deprecate, archive, or rename Markdown control-plane surfaces

## Canonical Runbooks

- [local_dev.md](./local_dev.md)
  - local environment and day-to-day developer workflow
- [pipeline_run.md](./pipeline_run.md)
  - canonical pipeline / demo execution flow
- [release_send_pdf.md](./release_send_pdf.md)
  - release-gated PDF send flow
- [doc_gardening.md](./doc_gardening.md)
  - markdown truthfulness closeout flow for rename / deprecate / archive / supersede work
