## Strict Force Definitions

The new board uses only two force families.

Penalty force:

- rigid-collision force only
- the force generated by the rigid-contact penalty / particle-vs-shape contact
  path on that cloth node
- saved and rendered per colliding node

Total force:

- full resultant force entering the particle update, expressed in force units
- defined as:
  - `f_internal_total + f_external_total + mass * gravity_vec`
- this includes spring/internal cloth force, rigid-collision penalty force, and
  gravity
- drag correction is not included in this force sum when drag is implemented as
  a post-step velocity correction instead of a force accumulation term

The exact force definition used by the promoted run must be written into the
run-local summary.

## Acceptance Criteria

### A. Detector KPI

- The first-collision detector is deterministic and saved.
- The detector records both:
  - geometry contact
  - force-active rigid contact
- The displayed node set matches the force-active collision set.
- Another agent can replay the board from saved detector artifacts without
  rerunning the simulation.

### B. Force-Semantics KPI

- Penalty panels show rigid-collision force only.
- Total panels show the full resultant force defined above.
- Arrow direction matches force direction.
- Arrow length is proportional to force magnitude.
- Color encodes magnitude with red as high force.
- One shared color scale is used across both penalty panels.
- One shared color scale is used across both total panels.

### C. Visualization KPI

- The main board is not trigger-only.
- The main board is not top-k-only.
- Cloth and rigid target remain readable through:
  - pre-contact
  - first contact
  - penetration growth / contact development
  - one second after first collision
- The contact ROI is readable without sacrificing scene context.
- A visible color bar / legend is present.
- A viewer can understand the difference between `box` and `bunny` without
  reading code.

### D. Contract / QA KPI

- The promoted run follows the experiment directory contract.
- Validators pass.
- Task/status/current-status docs reflect the reopened state.
